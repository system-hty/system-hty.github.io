GitHub OAuth配置问题

redirect_uri不匹配错误解决方案

您遇到了GitHub OAuth配置问题：redirect_uri不匹配。

这通常是因为GitHub OAuth应用设置中的回调URL与代码中使用的URL不一致。

解决方案

1

登录GitHub开发者设置

访问 GitHub OAuth Apps 设置，选择您的应用程序（SYSTEM-HTY）。

2

检查回调URL设置

确保"Authorization callback URL"字段设置为：

https://system-hty.github.io/bbs

注意：URL必须完全匹配，包括协议(https)和路径(/bbs)。

3

验证客户端ID

确保您的代码中使用正确的客户端ID：

Ov23liv4elLYx7XVnIQ3
4

检查OAuth请求URL

确保您的登录按钮指向正确的URL：

https://github.com/login/oauth/authorize?client_id=Ov23liv4elLYx7XVnIQ3&redirect_uri=https://system-hty.github.io/bbs&scope=user:email
5

处理回调

确保您有一个处理OAuth回调的页面位于：

https://system-hty.github.io/bbs

这个页面应该能够处理GitHub返回的授权码，并与服务器交换访问令牌。
