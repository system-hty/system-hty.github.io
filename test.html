解决GitHub登录redirect_uri问题

按照以下步骤配置您的GitHub OAuth应用

问题描述

当尝试使用GitHub登录时，您遇到了redirect_uri不匹配的错误。这是因为GitHub OAuth应用的回调URL配置不正确。

您当前的配置是：

file:///C:/web/github/my-personal-website/test.html

GitHub OAuth要求使用HTTP/HTTPS协议的回调URL，不能使用本地文件路径。

解决方案

1
登录GitHub账户
访问GitHub并登录您的账户
登录GitHub
2
访问开发者设置
转到Settings → Developer settings → OAuth Apps
打开开发者设置
3
编辑您的OAuth应用
找到并编辑您的OAuth应用
4
设置授权回调URL
将授权回调URL设置为以下值：
https://system-hty.github.io/login.html

如果您在本地开发，可以使用：

http://localhost:8080/login.html
5
保存更改
点击保存按钮应用更改
